W tym kroku połączymy dwa światy: surowy obraz z Twojej kamery oraz inteligencję detektora dłoni. Kluczem do sukcesu jest poprawna, asynchroniczna inicjalizacja obu źródeł danych.

### Inicjalizacja detektora

Ponieważ model musi pobrać swoje "wiedzę" (wagi sieci) z internetu, operacja ta musi być asynchroniczna. Nie chcemy blokować interfejsu użytkownika podczas ładowania kilkunastu megabajtów danych.

```ts showLineNumbers
const init = async () => {
  const hp = window.handPoseDetection;
  const model = hp.SupportedModels.MediaPipeHands;
  
  // Tworzymy instancję detektora
  detector = await hp.createDetector(model, modelConfig);
};
```

### Dostęp do sprzętu

Aby pobrać obraz, korzystamy z API `navigator.mediaDevices.getUserMedia`. Musimy określić pożądaną rozdzielczość - standardowe $640 \times 480$ jest idealnym komprosem między precyzją a obciążeniem GPU.

```ts
const stream = await navigator.mediaDevices.getUserMedia({
  video: { width: 640, height: 480 },
  audio: false, // Nie potrzebujemy dźwięku do wizji
});

if (videoRef.current) {
  videoRef.current.srcObject = stream;
}
```

> [!IMPORTANT]
> Przeglądarka zapyta użytkownika o zgodę na dostęp do kamery. Programista musi obsłużyć przypadek, w którym użytkownik odmówi (tzw. `PermissionDeniedError`).

---

### Gotowość do pracy

Zanim zaczniemy przesyłać klatki do modelu AI, musimy upewnić się, że wideo zostało faktycznie załadowane do pamięci karty graficznej. Sprawdzamy stan `readyState`:

```ts
// readyState 2 (HAVE_CURRENT_DATA) to minimum do analizy
const isVideoReady = videoRef.current?.readyState >= 2;
```

#### Faza "Rozgrzewki" (Warm-up)
Warto wiedzieć, że pierwsza klatka analizowana przez model na GPU trwa zazwyczaj znacznie dłużej (nawet 500-1000ms) niż kolejne. Jest to spowodowane koniecznością skompilowania shaderów WebGL. Po tej fazie, system osiąga docelową prędkość.
