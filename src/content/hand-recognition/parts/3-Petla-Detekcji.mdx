Mamy model i mamy wideo, ale system jeszcze "stoi". Potrzebujemy serca aplikacji — pętli, która będzie cyklicznie pobierać klatkę, wysyłać ją do GPU i odbierać wyniki.

### Synchronizacja z ekranem (60 FPS)

Nowoczesne monitory odświeżają obraz co ok. 16ms ($1/60$ sekundy). Jeśli nasza pętla będzie działać szybciej lub wolniej, użytkownik odczuje "rwanie" obrazu. Rozwiązaniem jest `requestAnimationFrame` — API, które prosi przeglądarkę o wywołanie funkcji dokładnie przed kolejnym odświeżeniem ekranu.

```ts showLineNumbers
const detect = async () => {
  // 1. Sprawdzamy gotowość sprzętu
  if (!videoRef.current || !detector) return;
  if (videoRef.current.readyState < 2) {
    requestAnimationFrame(detect);
    return;
  }

  // 2. Wykonujemy analizę (AI)
  const hands = await detector.estimateHands(videoRef.current);
  
  // 3. Wizualizujemy wyniki
  drawHands(hands, canvasRef.current);

  // 4. Planujemy następny cykl
  requestAnimationFrame(detect);
};
```

### Co robi `estimateHands`?

To najbardziej "pracowita" linia kodu. Pod maską wykonuje ona:
1.  **Konwersję klatki**: Zmianę pikseli z `<video>` na tensory.
2.  **Przesył danych**: Transfer tensorów do pamięci VRAM karty graficznej.
3.  **Inferencję**: Przepuszczenie danych przez setki warstw splotowych (CNN).
4.  **Ekstrakcję**: Przetłumaczenie aktywności neuronów na współrzędne 21 punktów.

> [!IMPORTANT]
> Ponieważ `estimateHands` jest asynchroniczne (`await`), nasza pętla nie blokuje głównego wątku przeglądarki. Interfejs pozostaje responsywny, nawet gdy GPU jest mocno obciążone.

### Zarządzanie cyklem życia

Bardzo ważne jest, aby pętla przestała działać, gdy użytkownik opuści stronę lub zamknie moduł. W przeciwnym razie kamera będzie aktywna w tle, marnując energię.

```ts
// Mechanizm "sprzątania" w React
useEffect(() => {
  return () => {
    if (animationFrameId) cancelAnimationFrame(animationFrameId);
    stopCameraStream();
  };
}, []);
```
