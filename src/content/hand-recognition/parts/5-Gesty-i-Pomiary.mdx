W ostatnim kroku nauczymy się tego, co czyni rozpoznawanie dłoni tak potężnym narzędziem: przekształcenia punktów w interaktywne **gesty**. Dzięki danym 3D, możemy mierzyć rzeczywiste odległości między palcami.

### Matematyka odległości 3D

Mimo że obraz jest płaski, model AI dostarcza nam współrzędną $z$ dla każdego punktu. Pozwala to na obliczenie odległości euklidesowej w przestrzeni trójwymiarowej:

$$ d = \sqrt{(x_2 - x_1)^2 + (y_2 - y_1)^2 + (z_2 - z_1)^2} $$

```ts
const getDistance = (a, b) => {
  return Math.sqrt((a.x - b.x)**2 + (a.y - b.y)**2 + (a.z - b.z)**2);
};
```

### Detekcja gestu szczypania (Pinch)

Gesty "szczypania" są najpopularniejszym sposobem sterowania interfejsami bezdotykowymi (np. w systemach VR). Aby go wykryć, mierzymy dystans między:
- **Punktem 4**: Opuszka kciuka.
- **Punktem 8**: Opuszka palca wskazującego.

### Normalizacja i Clamping

Surowa odległość (np. w centymetrach) jest niewygodna dla programisty. Musimy ją sprowadzić do zakresu $[0, 1]$, gdzie `0` oznacza dotyk, a `1` maksymalne rozchylenie:

1.  **Mapowanie**: Przeliczamy dystans na wartość procentową.
2.  **Clamping**: Ograniczamy wynik, aby nie wyszedł poza zakres $[0, 1]$.

```ts
const normalized = (dist - min) / (max - min);
const clamped = Math.max(0, Math.min(1, normalized));
```

> [!IMPORTANT]
> Dzięki normalizacji, Twój kod sterujący suwakami lub przyciskami na ekranie będzie działał tak samo dobrze u kogoś z dużą i małą dłonią.

### Możliwości rozwoju

Wykorzystując tę samą logikę, możesz zbudować własną bibliotekę gestów:
- **Zaciśnięta pięść**: Odległość wszystkich opuszek od środka dłoni (punkt 9) jest mała.
- **Otwarta dłoń**: Wszystkie palce są maksymalnie oddalone od nadgarstka.
- **Wskazywanie**: Palec wskazujący jest wyprostowany, a pozostałe zgięte.

Właśnie zbudowałeś kompletny system interakcji oparty na Sztucznej Inteligencji!
